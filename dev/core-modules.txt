# Pure-core module allowlist (dev/PLAN.md §0/D5).

# The HARD coverage-floor tier: pure, data-in/data-out modules where "cover every branch" and
# "write a real behavior test" are the same act — so a hard floor here is free of the
# test-padding failure mode and is the highest-value place to harden (a wrong SQL-context
# silently corrupts every autocomplete suggestion).
#
# One repo-relative path prefix per line (matches files whose cobertura filename starts with
# it); blank lines and `#` comments ignored. Grows as pure-core modules land per phase.

# Post-5 UX redesign (Stage 1 foundation): the Simple-mode 5-pane query form's pure projector
# (composer) and parser (simplifier). The composer trims pane texts, applies the documented
# fallbacks (empty SELECT -> `*`, empty LIMIT -> default, `0`/`all` -> omit, non-numeric -> error),
# and emits the dispatched `SELECT … FROM t [WHERE …] [GROUP BY …] [ORDER BY …] LIMIT n`. The
# simplifier reads the shared SQL lexer's tokens, validates the restricted single-`SELECT FROM t`
# grammar (rejects JOIN/CTE/subquery/HAVING/non-`t` table/multi-statement), and slices the original
# source into per-pane substrings. Both are pure data-in/data-out — a wrong fallback or a wrong
# clause split corrupts the dispatched SQL silently, every branch is a real behavior case, so they
# earn the hard floor.
src/app/query_form/composer.rs
src/app/query_form/simplifier.rs

# Post-5 UX redesign (Stage 1 foundation): the per-pane Simple-mode autocomplete adapter. Pure
# data-in/data-out — given (pane, pane_text, cursor, &Schema, &OperatorTable, &ValueCache) it
# synthesizes the governing clause prefix, classifies the cursor context, and merges the
# ranked column candidates with the ASC/DESC tail (filtered by the active partial). Every
# Simple-mode keystroke flows through this adapter (the App's `autocomplete_app::refresh_simple_mode`
# calls `pane_suggestions` and the value-fetch path calls `pane_context`); a wrong synthesized
# prefix, a wrong cursor offset, or a wrong slot detection silently corrupts every Simple-mode
# suggestion — the same failure mode that earned `clause_context.rs` its hard floor.
src/autocomplete/pane_context.rs
